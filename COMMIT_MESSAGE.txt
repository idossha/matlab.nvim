fix: resolve critical bugs and improve error handling

This commit addresses multiple critical bugs and adds comprehensive error
handling throughout the codebase.

Critical bug fixes:
- Fix filetype detection bug in commands.lua (was checking vim.bo.syntax
  instead of vim.bo.filetype, causing "Not a MATLAB script" errors)
- Fix pane_exists() in tmux.lua to properly validate pane existence using
  list-panes instead of has-session
- Fix incorrect Linux detection condition (operator precedence issue with
  'not' operator)
- Replace workspace toggle() command that relied on GUI-only 'workspace'
  command with 'whos' to work in -nodesktop mode

Error handling improvements:
- Add comprehensive file operation validation in workspace.lua (save/load)
- Add buffer modifiability checks before write operations
- Add pcall error handling for all vim.cmd('write') calls
- Add file existence validation before operations
- Add proper error messages for MATLAB server not running
- Add filename validation and automatic .mat extension handling

Code quality improvements:
- Create centralized notification system in new utils.lua module
- Remove code duplication across init.lua, tmux.lua, and ftplugin/matlab.lua
- Consolidate logging functionality into single reusable utility
- Improve consistency in error messages and logging format
- Add proper tmux.exists() checks in cell execution functions

All changes maintain backward compatibility and improve user experience
with more informative error messages and robust error handling.
